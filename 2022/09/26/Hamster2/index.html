上面是Hamster的主体，由于整个框架是基于PO（Page Object）来完成的，所以目录中的分层也是基于此

框架主体上分为：核心层、页面层、逻辑层、用例层、工具库、配置文件以及其他辅助工具，下面我将对每一个模块进行说明

核心层

核心层包含了两个部分，第一部分是设备驱动，第二部分是Hamster

设备驱动

我们先来讲讲设备驱动（Driver），由于Android和IOS连接方式不一致，所以，我分成了两个启动装置，如下：

# 启动装置（Android）
def connect_device(self, device):
    # 获取设备ID
    device_id = self.read_ini.get_value(device)
    # 连接设备
    return u2.connect(device_id)

# 启动装置（IOS）
def connect_device_ios(self, device):
    # 获取设备信息
    device_info = eval(self.read_ini.get_value(device))
    # 强制启动wda-tidevice
    wda._start_wda_xctest(device_info['id'])
    # 连接设备
    return wda.USBClient(device_info['id'], port=device_info['port'])

从配置文件中读取设备号，再通过uiautomator2和wda来连接设备

当然，为了更加便捷，我设计了设备池（POD），有了POD，无需配置设备号，设备号会直接从设备池中获取，下面展示的是获取Android设备

def pool_of_devices(self):
    """ 设备池 """
    cmd = r'adb devices'
    res = subprocess.Popen(cmd, stdout=subprocess.PIPE, shell=True)
    res.wait()
    out = res.stdout.readlines()
    devices = list()
    for i in (out)[1:-1]:
        device = str(i).split("\\")[0].split("'")[-1]
        devices.append(device)
    return devices

当设备连接后，会返回一个设备实例，以供我们进行操作，到这里，我们与手机设备的通信就连接成功了

Hamster

设备连接成功后返回的实例将传给Hamster，实例化Hamster，这里采用单例模式

Hamster在构造方法的时候，会初始化等待时间和延迟时间。上篇说过Hamster是对uiautomator2的二次封装，那么核心层的Hamster就是它的体现

做UI自动化测试最重要的是获取元素，Hamster亦是如此。获取元素方法中，支持多种参数类型，并且为了将元素定位信息抽离用例本身，这里统一配置成配置文件，这样做也是为了之后的平台化

为了使页面层写起来更流畅，元素定位时，可以传入不同类型的参数，支持：dict、str和list

if isinstance(location, dict):
    element = EleAnalysis(location['fileName'], location['node'])
    target = element.get_parameter(key)
elif isinstance(location, str):
    target = {location: key}
elif isinstance(location, list) and isinstance(key, list):
    target = {}
    for i in range(location.__len__()):
        target[location[i]] = key[i]
else:
    raise Exception('location or key 参数错误')

Hamster中还包含了多种元素操作事件，如：单击事件、广播输入事件、指定输入事件、获取文本信息、屏幕滑动、元素拖动、获取屏幕尺寸、点亮屏幕、锁屏判断、解锁事件、图片判断、文字判断等多种事件，基本上满足了多种情况下的应用需求

上面提到的事件中，除了图片识别判断和文字识别判断外，其他的基本都是对uiautomator2和wda的二次封装，大家可以去uiautomator2和wda中详细了解，关于图片识别和文字识别我将在之后单独和大家分享

页面层

这一层是获取页面元素以及进行元素操作的模块

页面层分为两个部分，第一部分是元素定位配置，第二部分是元素操作处理

通过页面元素获取工具（WEditor）获取到元素关键信息后，就可以写入元素定位配置文件中， 之后传入Hamster实例，实例化Page类，从而驱动操作事件

from Utils.CatchError import catcherr


class WeChatPage:
    def __init__(self, launch, device_type):
        self.element = launch
        self.dt = device_type

工具库

工具库中包含了读取INI配置文件、日志处理、元素解析、图片识别、OCR识别以及捕获异常

图片识别和OCR识别之后单独分享，这里只说下捕获异常，写这个捕获异常是为了更加精确的定位元素操作的错误，我是以装饰器的方式完成的

def catcherr(func):
    def wapper(*args, **kwargs):
        try:
            res = func(*args, **kwargs)
        except:
            # 抛出异常，信息为方法注释信息
            raise Exception('【{}】操作错误'.format(func.__doc__))
        return res
    return wapper

通过上面的装饰器加上日志信息，我能快速准确定位问题

其他的工具都比较简单了我的源码中也写了详细的注释以供参考
