给自己的GitHub主页放只贪吃蛇吧

最近又有时间去逛逛GitHub，去寻找新鲜的东西，还别说，每次都能发现有趣的东西😄

这次能我发现了一只能吃Commit的贪吃蛇，就像下面这样的👇

今天就来教大家如何在自己的GitHub主页添加一只有趣的贪吃蛇

Step1

在Github上新建一个与自己账号同名的公共仓库（Public Repository ），创建时，选择创建README文件

Step2

创建完成后，点击Add File，选择Create new file

Step3

进入创建文件页面后，在文件名出输入.github/workflows/GenerateSnk.yml，之后内容处输入以下内容并保存（这里展示的是我主页配置信息）

name: generate snk

on:
  # run automatically every 6 hours
  schedule:
    - cron: "0 0 * * *" 
  
  # allows to manually run the job at any time
  workflow_dispatch:
  
  # run on every push on the master branch
  push:
    branches:
    - master

jobs:
  generate:
    runs-on: ubuntu-latest
    timeout-minutes: 10
    
    steps:
      # generates a snake game from a github user (<github_user_name>) contributions graph, output a svg animation at <svg_out_path>
      - name: generate github-contribution-grid-snake.svg
        uses: Platane/snk/svg-only@v2
        with:
          github_user_name: ${{ github.repository_owner }}
          outputs: |
            dist/github-contribution-grid-snake.svg
            dist/github-contribution-grid-snake-dark.svg?palette=github-dark
      # push the content of <build_dir> to a branch
      # the content will be available at https://raw.githubusercontent.com/<github_user>/<repository>/<target_branch>/<file> , or as github page
      - name: push github-contribution-grid-snake.svg to the output branch
        uses: crazy-max/ghaction-github-pages@v2.6.0
        with:
          target_branch: master
          build_dir: dist
          keep_history: true
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

Step4

保存文件后，返回仓库并刷新页面，你就会发现仓库中会新增两个文件

Step5

编辑README.md文件，并增加生成的贪吃蛇，因为是svg 文件，Markdown添加方式如下：

<img align="center" src="https://raw.githubusercontent.com/你的GithubID/你的GithubID/master/github-contribution-grid-snake-dark.svg" />

保存完成后就可以查看效果了

整体配置完成还是比较快比较容易的，希望能对大家有所帮助
